# 리포트 검증 규칙 (report-validation-rules)

## 목적

리포트가 구조 규칙을 준수하는지 검증하여 합격/불합격을 판정한다.

---

## 검증 체크리스트

### 필수 섹션 존재 여부

| # | 검증 항목 | 합격 기준 |
|---|-----------|-----------|
| 1 | 요약 섹션 | 1~2줄의 요약 문장 존재 |
| 2 | ASIS/TOBE 비교표 | 테이블 형식, 최소 1행 이상 |
| 3 | 왜 (동기) 섹션 | 섹션 헤더 + 간략 요약 존재 |
| 4 | 무엇을 (대상/범위) 섹션 | 섹션 헤더 + 간략 요약 존재 |
| 5 | 어떻게 (방법/계획) 섹션 | 섹션 헤더 + 간략 요약 존재 |

### 왜/무엇을/어떻게 완전성

| # | 검증 항목 | 합격 기준 |
|---|-----------|-----------|
| 6 | 왜 — 간략 | 2~3줄 이내의 요약 존재 |
| 7 | 왜 — 확장 | `<details>` 태그 내 상세 내용 존재 |
| 8 | 무엇을 — 간략 | bullet point 형태의 포함/제외 항목 존재 |
| 9 | 무엇을 — 확장 | `<details>` 태그 내 상세 내용 존재 |
| 10 | 어떻게 — 간략 | 번호 목록 형태의 주요 단계 존재 |
| 11 | 어떻게 — 확장 | `<details>` 태그 내 상세 내용 존재 |

### 간략+확장 구조 준수

| # | 검증 항목 | 합격 기준 |
|---|-----------|-----------|
| 12 | 간략 독립성 | 간략만 읽어도 핵심 내용 파악 가능 |
| 13 | 확장 형식 | `<details><summary>상세</summary>` 형식 사용 |
| 14 | 간략 분량 | 각 간략 섹션이 3줄 이내 |

### 입력 데이터 반영

| # | 검증 항목 | 합격 기준 |
|---|-----------|-----------|
| 15 | ASIS 요소 반영 | answer-analysis의 ASIS 요소가 비교표에 모두 포함 |
| 16 | TOBE 요소 반영 | answer-analysis의 TOBE 요소가 비교표에 모두 포함 |
| 17 | 추정 표시 | 확인되지 않은 요소에 `(추정)` 태그 존재 |
| 18 | 누락 요소 없음 | answer-analysis 산출물 대비 리포트에 누락된 요소 없음 |

---

## 판정 기준

| 판정 | 조건 |
|------|------|
| **합격** | 18개 항목 중 불합격 항목이 0개 |
| **조건부 합격** | 불합격 항목이 1~2개이며, 모두 확장(7, 9, 11) 관련 |
| **불합격** | 위 조건 외 모든 경우 |

---

## 산출물 형식

```markdown
## 검증 결과

**판정: {합격/조건부 합격/불합격}**

| # | 검증 항목 | 결과 | 비고 |
|---|-----------|------|------|
| 1 | 요약 섹션 | PASS/FAIL | ... |
| 2 | ... | ... | ... |

## 불합격 사유 (해당 시)

- 항목 #N: {불합격 사유}
- ...
```
