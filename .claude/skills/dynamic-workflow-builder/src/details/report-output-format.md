# 리포트 출력 형식 (report-output-format)

## 목적

답변 분석 결과를 ASIS/TOBE 비교 -> 왜(동기) -> 무엇을(대상/범위) -> 어떻게(방법/계획) 구조의 리포트로 작성한다.

---

## 리포트 전체 구조

```
1. 요약 (1~2줄)
2. ASIS/TOBE 비교표
3. 왜 (동기)
4. 무엇을 (대상/범위)
5. 어떻게 (방법/계획)
```

---

## 섹션별 작성 규칙

### 1. 요약

- 1~2줄로 이 리포트가 다루는 핵심 내용을 기술
- "무엇을 왜 어떻게 하는지"를 한 문장으로 압축

### 2. ASIS/TOBE 비교표

| 항목 | ASIS (현재) | TOBE (목표) |
|------|-------------|-------------|
| {비교 축} | {현재 상태} | {목표 상태} |

- 비교 축은 answer-analysis에서 추출한 ASIS/TOBE 요소를 기준으로 구성
- 각 행이 하나의 변경 포인트를 나타냄
- "추정" 태그가 붙은 요소는 비교표에서 `(추정)` 표시

### 3. 왜 (동기)

**간략**: 2~3줄로 핵심 동기 요약

<details>
<summary>상세</summary>

- 변경의 배경과 맥락
- 현재 상태의 문제점 또는 개선 필요성
- 이해관계자의 기대 또는 요구사항

</details>

### 4. 무엇을 (대상/범위)

**간략**: 포함/제외 항목을 bullet point로 나열

<details>
<summary>상세</summary>

- 변경 대상의 구체적 범위
- 명시적으로 제외된 항목
- 의존성 및 영향 범위

</details>

### 5. 어떻게 (방법/계획)

**간략**: 주요 단계를 번호 목록으로 나열

<details>
<summary>상세</summary>

- 각 단계의 구체적 방법
- 일정 또는 순서 제약
- 리스크 및 대응 방안

</details>

---

## 간략/확장 구조 규칙

| 규칙 | 설명 |
|------|------|
| 간략 필수 | 왜/무엇을/어떻게 각 섹션에 반드시 간략 요약 존재 |
| 확장 선택적 | `<details>` 태그로 접을 수 있는 상세 내용 |
| 간략 분량 | 2~3줄 이내 |
| 확장 분량 | 제한 없으나 bullet point 위주 |
| 독립 가독성 | 간략만 읽어도 핵심 내용 파악 가능 |

---

## 작성 원칙

1. **근거 기반**: 모든 기술은 answer-analysis 산출물의 ASIS/TOBE 요소에 기반
2. **추정 명시**: 사용자 답변으로 확인되지 않은 요소는 `(추정)` 표시
3. **산문 최소화**: 표, bullet point, 번호 목록 위주로 작성
4. **누락 금지**: answer-analysis에서 추출된 ASIS/TOBE 요소가 리포트에 모두 반영되어야 함
