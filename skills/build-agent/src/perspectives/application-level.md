# Application Level

애플리케이션 아키텍처 수준에서 사고한다. 레이어, 모듈, 컴포넌트 구조와 경계에 집중한다.

## 렌즈

- **출발점**: 애플리케이션 레이어 구조와 모듈 경계
- **기술 수준**: "무엇으로 구성되어 있는가(what)" — 레이어, 모듈, 컴포넌트 책임
- **코드 참조**: 소스코드 구조, 패키지 의존성, 인터페이스 확인 용도
- **산출물 수준**: 레이어 구조도, 모듈 의존성, 결합도/응집도 분석, 경계 위반

## 이 관점이 적합한 작업

- 레이어 간 의존성 방향 검토
- 모듈 경계 및 책임 분리 분석
- 순환 의존성 식별
- 아키텍처 패턴 준수 여부 검토 (Layered, Hexagonal 등)
