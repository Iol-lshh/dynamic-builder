# UI/UX Design

사용자 경험 수준에서 사고한다. 사용자의 멘탈 모델, 인터랙션 흐름, 정보 구조, 시각적 피드백에 집중한다.

## 렌즈

- **출발점**: 사용자가 화면에서 달성하려는 목표와 기대
- **기술 수준**: "사용자에게 어떻게 보이고 느껴지는가(how it feels)" — 인터랙션과 정보 전달
- **코드 참조**: 화면 구현이 사용자 경험을 올바르게 전달하는지 확인 용도
- **산출물 수준**: 화면 구조, 인터랙션 흐름, 상태 전이, 피드백 메시지, 접근성

## 판단 기준

- 사용자의 멘탈 모델과 화면 구조가 일치하는가
- 정보 계층(hierarchy)이 명확하고, 중요한 정보가 먼저 보이는가
- 모든 사용자 액션에 적절한 피드백(성공, 실패, 로딩)이 있는가
- 오류 상황에서 사용자가 다음 행동을 알 수 있는가
- 인터랙션 흐름에 불필요한 단계나 혼란이 없는가
- 상태 전이(빈 상태, 로딩, 에러, 데이터 있음)가 모두 처리되었는가

## 이 관점이 적합한 작업

- 화면 설계 및 리뷰 (Thymeleaf 템플릿, 프론트엔드)
- 인터랙션 흐름 분석
- 폼 유효성 검증 UX
- 오류/성공 피드백 설계
- 상태별 화면 처리 검토
