# Grouping Review Guide

그룹화 결과(두 배정안)의 품질을 채점하는 기준과 score 산출 방식을 정의한다.

## 채점 대상

optimize-grouping 스텝의 산출물에 포함된 두 배정안을 각각 독립적으로 채점한다.

- 배정안 1: 부분 최적화 (Local Optimization)
- 배정안 2: 전체 최적화 (Global Optimization, Maximin)

## 평가 항목 및 배점

### 부분 최적화 평가 (100점 만점)

| # | 항목 | 배점 | 평가 기준 |
|---|------|------|-----------|
| 1 | 최강 팀 점수 극대화 | 30 | 최고 팀 총점이 이론적 최댓값에 근접하는가. compute-team-scores의 조합별 점수와 비교하여 최적 조합이 선택되었는지 검증 |
| 2 | 팀 내 보완성 반영 | 25 | analyze-complementarity의 보완성 매트릭스가 배정에 실제로 반영되었는가. 정량 점수만으로 배정하지 않고 질적 보완성을 고려했는지 확인 |
| 3 | 배정 근거 추적 가능성 | 20 | 각 팀의 배정 근거(정량/정성)가 명확히 기술되어, 왜 이 조합이 선택되었는지 역추적 가능한가 |
| 4 | 경계 사례 처리 | 15 | 점수가 유사한 대안 조합이 있을 때 선택 논리가 합리적인가. 동점 또는 근소 차이 상황의 판단 근거가 명시되어 있는가 |
| 5 | 출력 형식 준수 | 10 | grouping-output-format.md에 정의된 형식을 준수하는가 |

### 전체 최적화 평가 (100점 만점)

| # | 항목 | 배점 | 평가 기준 |
|---|------|------|-----------|
| 1 | 최약 팀 점수 극대화 (maximin) | 30 | 모든 팀 중 최솟값이 가능한 한 높은가. 대안 배정안 대비 최약 팀 점수가 실제로 최대화되었는지 검증 |
| 2 | 팀 간 균형 | 25 | 팀 총점의 편차가 최소화되었는가. 특정 팀만 과도하게 약하거나 강하지 않은가 |
| 3 | 팀 내 보완성 반영 | 20 | 부분 최적화와 동일 기준. 전체 균형 추구 과정에서 질적 보완성이 희생되지 않았는가 |
| 4 | 배정 근거 추적 가능성 | 15 | 부분 최적화와 동일 기준 |
| 5 | 출력 형식 준수 | 10 | grouping-output-format.md에 정의된 형식을 준수하는가 |

## score 산출 방식

### 개별 배정안 점수 계산

각 배정안의 점수는 위 5개 항목의 합산이다 (0-100).

### 최종 score 규칙

```
최종 score = min(부분최적화 점수, 전체최적화 점수)
```

두 배정안의 품질 점수 중 **낮은 값**을 최종 score로 사용한다. 이는 한쪽 배정안만 우수하고 다른 쪽이 부실한 결과를 방지하기 위함이다.

### retry 조건과의 관계

- 워크플로우의 retry 조건: `score < 80`
- 따라서 두 배정안 모두 80점 이상이어야 retry 루프를 통과한다.
- 한 배정안이 90점이라도 다른 배정안이 75점이면 score = 75로 retry가 발생한다.

```markdown
|------|------|------|

| 팀 내 보완성 반영 | /25 | |
| 배정 근거 추적 가능성 | /20 | |
| 경계 사례 처리 | /15 | |
| 출력 형식 준수 | /10 | |
| **소계** | **/100** | |

## 전체 최적화 채점

(동일 형식)

## 최종 score

- 부분 최적화: {X}점
- 전체 최적화: {Y}점
- **score = min({X}, {Y}) = {Z}**
```

## 주의사항

- 두 배정안은 서로 다른 전략이므로, 동일한 배정이 나오면 감점 사유이다.
- compute-team-scores의 조합별 점수 데이터를 참조하여 배정의 정량적 최적성을 검증한다.
- analyze-complementarity의 보완성 매트릭스를 참조하여 질적 판단의 반영 여부를 확인한다.
