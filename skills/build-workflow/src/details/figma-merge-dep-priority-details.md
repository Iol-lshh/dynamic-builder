# figma-merge-dep-priority-details.md

스텝: merge-dep-priority
에이전트: scope-reduction-reconciler

## 목적

의존관계 분석과 중요도 분석을 통합하여 우선순위가 반영된 최종 백로그를 생성한다.

## 입력

- `dependencies.md`: analyze-dependencies의 산출물
- `priority.md`: analyze-priority의 산출물

## 통합 규칙

### 1. 우선순위 산정

의존관계와 중요도를 결합하여 실행 우선순위를 결정한다:

- **중요도 우선**: 동일 의존 레벨에서는 중요도가 높은 항목이 먼저
- **의존관계 제약**: 선후행/블로커 관계에 의해 실행 순서가 강제되는 경우, 중요도보다 의존관계가 우선
- **병렬 가능 그룹**: 독립 항목 + 의존 관계 없는 항목을 병렬 실행 가능 그룹으로 묶는다

### 2. 실행 순서 결정

1. 블로커 관계의 시작점(의존 없는 항목) 중 중요도 높은 순으로 정렬
2. 선후행 관계를 반영하여 위상 정렬
3. 동일 레벨 내에서 중요도 순 정렬
4. 공유 자원 충돌이 있는 항목은 동일 스프린트 배치를 피한다

### 3. 스프린트 배치 제안

- 위상 정렬 결과를 기반으로 스프린트 단위 배치를 제안한다
- 각 스프린트의 예상 항목 수는 3-7개로 제한한다

## 산출물 형식

```markdown
# 우선순위 반영 최종 백로그

## 실행 순서

| 순위 | 항목 | 중요도 | 의존 선행 | 스프린트 |
|------|------|--------|-----------|----------|
| 1 | BL-xxx | Critical | 없음 | Sprint 1 |
| 2 | BL-yyy | High | BL-xxx | Sprint 1 |
| ... |

## 스프린트 배치

### Sprint 1
{항목 목록, 병렬 실행 가능 그룹 표시}

### Sprint 2
...

## 의존관계 요약
{dependencies.md의 의존관계 맵 포함}

## 중요도 요약
{priority.md의 항목별 중요도 포함}

## 각 백로그 항목 (우선순위 순)

### BL-{번호}: {제목}
- **스코프**: AS-IS / MINIMUM / DEFER
- **중요도**: Critical / High / Medium / Low
- **의존관계**: {선행/후행 항목}
- **스프린트**: Sprint N
- **설명**: ...
- **구현 힌트**: ...
```

## 제약

- 백로그 항목의 내용(설명, 구현 힌트, 스코프)을 수정하지 않는다
- 의존관계와 중요도 정보를 추가만 한다
- 모든 항목이 실행 순서에 포함되어야 한다 (누락 금지)
