# 통합 문서 출력 형식

## 문서 구조

최종 산출물은 아래 6개 섹션으로 구성된 단일 마크다운 문서이다.

### 1. 세션 메타데이터

```markdown
# 세션 컨텍스트 요약

| 항목 | 값 |
|------|-----|
| 날짜 | YYYY-MM-DD |
| 목표 | 세션의 주요 목표 |
| 브랜치 | 작업 브랜치명 |
| 최종 상태 | 완료 / 진행 중 / 중단 |
| 미완료 사항 | 남은 작업 목록 (있는 경우) |
```

### 2. 산출물 목록

ctx-artifact-inventory 결과를 기반으로 작성한다.
- 경로, 유형, 상태, 설명을 테이블로 정리

### 3. 스텝 분류표

ctx-step-classification 결과를 기반으로 작성한다.
- 산출물별로 도출 과정의 스텝들을 정리

### 4. 역방향 최적 과정 요약

**최종 결과에서 출발하여** 각 스텝이 왜 필요했는지 역추적한다.

#### 작성 규칙

- **시작점**: 최종 산출물 (가장 중요한 결과물)
- **역추적 방향**: 최종 산출물 → 이를 만들기 위해 필요했던 중간 산출물 → 그 중간 산출물을 만들기 위한 입력 → ... → 세션 시작
- **깊이**: 세션 시작 시점까지
- **각 스텝 서술**: "X를 만들기 위해 Y가 필요했고, Y를 얻기 위해 Z를 수행했다"
- **불필요 스텝 식별**: 시행착오, 되돌린 작업, 우회 경로를 식별하고 "최적 경로에서는 이 스텝을 건너뛸 수 있다"고 명시
- **최적 경로 요약**: 불필요 스텝을 제거한 최단 경로를 별도로 정리

#### 출력 형태

```markdown
## 역방향 최적 과정 요약

### 최종 결과
{최종 산출물 설명}

### 역추적
1. **{최종 산출물}** ← {이를 위해 필요했던 것}
2. **{중간 산출물}** ← {이를 위해 필요했던 것}
...

### 최적 경로
{불필요 스텝을 제거한 최단 과정}
```

### 5. 스크립트 로그

ctx-script-collection 결과를 기반으로 작성한다.
- 실행 순서대로 명령어, 분류, 목적, 결과를 정리

### 6. 재개 지침

다음 세션에서 이 문서를 읽고 작업을 이어받을 때 알아야 할 핵심 정보:
- **현재 상태**: 어디까지 완료되었는가
- **다음 할 일**: 무엇을 이어서 해야 하는가
- **핵심 주의 사항**: 실수하기 쉬운 점, 알아두어야 할 제약 조건
- **관련 파일**: 핵심 파일 경로와 역할
- **의사결정 기록**: 왜 이 방식을 선택했는지 (대안과 비교)

## 통합 원칙

- 3개 분석 결과(artifact-inventory, step-classification, script-collection)를 **모두** 반영한다
- 분석 결과 간 불일치가 있으면 식별하고 조율한다
- 중복 정보는 가장 적합한 섹션에 한 번만 기록한다

## ctx-retrospect 전용 지침

ctx-retrospect는 ctx-review의 평가 피드백을 반영하여 문서를 재작성하는 스텝이다.

### 피드백 반영 우선순위

1. **감점 사유 해소**: 평가에서 지적된 감점 사유를 최우선으로 해결한다
2. **누락 보완**: 누락된 산출물, 스텝, 스크립트를 원본 분석 결과에서 찾아 추가한다
3. **논리 보강**: 역방향 요약의 논리적 단절이 지적되면 중간 연결 고리를 보충한다
4. **구조 유지**: 6개 섹션 구조는 변경하지 않는다. 섹션 내 내용만 개선한다

### 입력 활용

- ctx-synthesis: 이전 버전의 통합 문서 (기반)
- ctx-review: 평가 피드백 (개선 방향)
- ctx-artifact-inventory, ctx-step-classification, ctx-script-collection: 원본 분석 결과 (누락 보완용)
