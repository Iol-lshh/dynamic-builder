# 산출물 인벤토리 가이드

## 식별 범위

세션 내에서 생성, 수정, 삭제된 모든 산출물을 식별한다:
- 소스 코드 파일
- 문서 (마크다운, 텍스트)
- 설정 파일 (YAML, JSON, TOML)
- 데이터 파일
- 워크플로우/에이전트 템플릿

## 제외 대상

- 임시 파일, 캐시
- node_modules, .git 내부 파일
- 자동 생성된 lock 파일 (변경 의도가 없는 경우)

## 산출물 속성

각 산출물에 대해 다음 속성을 기록한다:

| 속성 | 설명 |
|------|------|
| 경로 | 파일의 전체 경로 |
| 유형 | 코드 / 문서 / 설정 / 데이터 / 템플릿 |
| 상태 | 생성 / 수정 / 삭제 |
| 설명 | 산출물의 목적과 내용 요약 (1줄) |

## 출력 형식

```markdown
## 산출물 목록

| # | 경로 | 유형 | 상태 | 설명 |
|---|------|------|------|------|
| 1 | path/to/file.md | 문서 | 생성 | ... |
```

## 완전성 확인

- 세션 대화에서 언급된 파일과 목록을 교차 검증한다
- Write/Edit 도구로 생성·수정한 파일이 누락되지 않았는지 확인한다
- Bash로 실행한 명령어의 출력 파일도 포함한다
