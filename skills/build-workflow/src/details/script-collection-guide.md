# 스크립트 수집 가이드

## 수집 대상

- Bash 명령어 (ls, mkdir, cp 등)
- Git 명령어 (git status, git diff, git commit 등)
- 패키지 관리 (npm, yarn, pip 등)
- 빌드/테스트 명령어 (npm run, jest, tsc 등)
- 코드 스니펫 (인라인으로 실행한 스크립트)
- 파이프라인 명령어 (연쇄 명령어)

## 분류 체계

| 분류 | 설명 | 예시 |
|------|------|------|
| 환경 설정 | 디렉토리 생성, 패키지 설치, 설정 변경 | mkdir -p, npm install |
| 정보 조회 | 상태 확인, 파일 검색, 로그 확인 | git status, ls, git log |
| 빌드/테스트 | 컴파일, 테스트 실행, lint | npm run build, jest |
| 배포/실행 | 서버 시작, 프로세스 관리, 스크립트 실행 | npm start, node script.js |
| 데이터 처리 | 파일 변환, 데이터 추출, 포맷팅 | jq, sed, awk |

## 수집 속성

| 속성 | 설명 |
|------|------|
| 순번 | 실행 순서 |
| 명령어 | 명령어 원문 (코드 블록) |
| 분류 | 위 분류 체계 중 하나 |
| 목적 | 이 명령어를 실행한 이유 |
| 결과 | 성공 / 실패 / 부분 성공 |
| 재사용 | 다음 세션에서 재사용 가능 여부 (Y/N) |

## 출력 형식

```markdown
## 스크립트 로그

| # | 명령어 | 분류 | 목적 | 결과 | 재사용 |
|---|--------|------|------|------|--------|
| 1 | `mkdir -p .local/output` | 환경 설정 | 출력 디렉토리 생성 | 성공 | Y |
```

## 주의사항

- Read, Glob, Grep 등 Claude 도구 호출은 스크립트가 아닌 도구 사용으로 분류하되, 맥락상 중요한 경우 포함한다
- 실패한 명령어도 포함한다 (디버깅 맥락 보존)
