# figma-validate-completeness-details.md

스텝: validate-completeness
에이전트: usecase-validator

---

## 목적

우선순위 반영 최종 백로그의 모든 항목에 의존관계와 중요도가 빠짐없이 배정되었는지 검증한다.

## 입력

- `backlog-prioritized.md`: merge-dep-priority의 산출물
- `backlog-final.md`: apply-advice의 산출물 (원본 백로그 항목 대조용)

## 검증 기준

### 1. 의존관계 배정 완전성

- `backlog-final.md`의 모든 항목이 `backlog-prioritized.md`의 의존관계 맵에 존재하는가
- 의존 관계가 없는 항목도 "독립 항목"으로 명시적 기술되어 있는가

### 2. 중요도 배정 완전성

- `backlog-final.md`의 모든 항목에 중요도(Critical/High/Medium/Low)가 배정되어 있는가
- 중요도 미배정 항목이 0개인가

### 3. 실행 순서 완전성

- 모든 항목이 실행 순서 테이블에 포함되어 있는가
- 모든 항목에 스프린트 배치가 지정되어 있는가

### 4. 의존관계-실행순서 정합성

- 선후행/블로커 관계가 실행 순서에 올바르게 반영되었는가 (선행 항목이 후행 항목보다 앞에 배치)
- 공유 자원 충돌 항목이 동일 스프린트에 배치되지 않았는가

## 피드백 유형 제한

이 스텝의 피드백은 **"배정 누락 목록"** 으로 제한한다.

허용되는 피드백:
- "BL-xxx에 의존관계가 미배정"
- "BL-xxx에 중요도가 미배정"
- "BL-xxx가 실행 순서에 누락"
- "BL-xxx가 스프린트에 미배치"
- "BL-xxx -> BL-yyy 선후행 관계가 실행 순서에 미반영"

허용되지 않는 피드백:
- 백로그 항목 내용의 품질 평가 (evaluate-backlog의 범위)
- 의존관계/중요도 판정의 타당성 평가 (분석 스텝의 범위)
- 스코프 판정의 적절성 평가 (apply-advice의 범위)

## 산출물 형식

```markdown
# 완전성 검증

## 점수 테이블

| 기준 | 점수 | 만점 |
|------|------|------|
| 의존관계 배정 완전성 | XX | 25 |
| 중요도 배정 완전성 | XX | 25 |
| 실행 순서 완전성 | XX | 25 |
| 의존관계-실행순서 정합성 | XX | 25 |
| **총점** | **XX** | **100** |

score: XX

## 배정 누락 목록

| 항목 | 누락 유형 | 설명 |
|------|-----------|------|
| BL-xxx | 의존관계 미배정 | ... |

(누락 없으면 "누락 항목 없음" 명시)

## 검증 요약

- 전체 항목 수: N
- 의존관계 배정 완료: N/N
- 중요도 배정 완료: N/N
- 실행 순서 포함: N/N
- 정합성 위반: N건
```

## 제약

- score 필드는 반드시 `score: XX` 형식으로 산출물에 기술한다 (retry 조건 파싱용)
- 피드백 유형을 "배정 누락 목록"으로 제한한다 (Phase 3 retry 범위 축소의 전제 조건)
- 백로그 항목을 수정하지 않는다 (검증만 수행)
