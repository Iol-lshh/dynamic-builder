# 리포트 채점 기준 (report-scoring-criteria)

## 목적

리포트의 품질을 5개 항목으로 채점하여 100점 만점 점수를 산출한다.

---

## 채점 항목

### 1. ASIS/TOBE 동기 명확성 (20점)

| 점수 | 기준 |
|------|------|
| 18~20 | ASIS/TOBE 비교표가 명확하고, 왜(동기) 섹션이 비교표와 논리적으로 연결됨. 변경의 필요성이 비교표만으로 즉시 파악 가능 |
| 14~17 | 비교표와 동기가 대부분 연결되나, 일부 비교 축의 동기가 불명확 |
| 10~13 | 비교표 또는 동기 중 하나가 부실. 변경 필요성을 파악하려면 추가 설명 필요 |
| 0~9 | 비교표와 동기가 단절되거나, 핵심 ASIS/TOBE 요소가 누락됨 |

### 2. 왜/무엇을/어떻게 완전성 (30점)

| 점수 | 기준 |
|------|------|
| 27~30 | 세 섹션 모두 간략+확장 구조로 완비. 간략만 읽어도 전체 그림 파악 가능. 확장이 간략을 구체적으로 보완 |
| 21~26 | 세 섹션 모두 존재하나, 일부 확장이 간략의 반복이거나 새로운 정보 부족 |
| 14~20 | 하나의 섹션이 실질적으로 비어 있거나, 간략이 핵심을 전달하지 못함 |
| 0~13 | 두 개 이상의 섹션이 부실하거나 누락 |

### 3. 사용자 답변 활용도 (20점)

| 점수 | 기준 |
|------|------|
| 18~20 | 사용자 답변의 핵심 내용이 리포트 전반에 반영됨. 답변에서 새로 확인된 사실이 ASIS/TOBE에 정확히 매핑됨 |
| 14~17 | 답변이 대부분 반영되었으나, 일부 답변 내용이 리포트에 누락됨 |
| 10~13 | 답변 반영이 피상적. 답변 내용이 리포트에 있으나 ASIS/TOBE와의 연결이 약함 |
| 0~9 | 답변 내용이 리포트에 거의 반영되지 않거나, 답변과 무관한 내용이 다수 |

### 4. 간결성 (15점)

| 점수 | 기준 |
|------|------|
| 13~15 | 간략 섹션이 2~3줄 이내로 핵심만 전달. 불필요한 반복이나 장황한 설명 없음. 표와 bullet point 위주 |
| 10~12 | 간략이 약간 길거나, 일부 중복 표현 존재. 전체적으로 읽기에 부담 없음 |
| 6~9 | 간략이 3줄을 초과하거나, 산문 비중이 높아 빠른 파악이 어려움 |
| 0~5 | 간략/상세 구분이 모호하거나, 전체가 산문 형태 |

### 5. 확장 구조 적합성 (15점)

| 점수 | 기준 |
|------|------|
| 13~15 | `<details>` 태그가 올바르게 사용됨. 확장 내용이 간략과 차별화된 상세 정보 제공. 접었을 때와 펼쳤을 때 모두 자연스러움 |
| 10~12 | 확장 구조가 존재하나, 일부 확장이 간략의 단순 반복 |
| 6~9 | 확장 구조가 불완전하거나, `<details>` 태그 사용이 부적절 |
| 0~5 | 확장 구조가 없거나, 구조가 깨져 있음 |

---

## 총점 산출

| 항목 | 배점 |
|------|------|
| ASIS/TOBE 동기 명확성 | 20 |
| 왜/무엇을/어떻게 완전성 | 30 |
| 사용자 답변 활용도 | 20 |
| 간결성 | 15 |
| 확장 구조 적합성 | 15 |
| **합계** | **100** |

---

## 산출물 형식

```markdown
## Score: {총점}/100

## 항목별 채점

| 항목 | 점수 | 근거 |
|------|------|------|
| ASIS/TOBE 동기 명확성 (20) | {점수}/20 | {구체적 근거} |
| 왜/무엇을/어떻게 완전성 (30) | {점수}/30 | {구체적 근거} |
| 사용자 답변 활용도 (20) | {점수}/20 | {구체적 근거} |
| 간결성 (15) | {점수}/15 | {구체적 근거} |
| 확장 구조 적합성 (15) | {점수}/15 | {구체적 근거} |

## 개선 필요 사항

- {항목}: {개선 권고}
```

## retry 판정

- **score >= 80**: 합격. 다음 단계(HITL #2)로 진행
- **score < 80**: 불합격. retry loop 재실행. 이 채점 결과가 다음 회차의 피드백으로 전달됨
