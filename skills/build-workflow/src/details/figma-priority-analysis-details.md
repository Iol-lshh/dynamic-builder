에이전트: domain-analyst

## 목적

확정된 백로그 항목별 중요도를 domain-analyst 관점(도메인 모델, 비즈니스 규칙)에서 산정한다.

## 입력

- `backlog-final.md`: apply-advice의 산출물 (확정된 백로그)

## 중요도 산정 4축

domain-analyst의 도메인 모델/비즈니스 규칙 렌즈를 활용하여 각 축을 평가한다.

### 1. 영향 범위 (Impact Scope)

- 해당 항목이 영향을 미치는 도메인 엔티티/경계 컨텍스트의 수
- 평가: 높음(3개 이상) / 중간(2개) / 낮음(1개)

### 2. 사용 빈도 (Usage Frequency)

- 도메인 이벤트 발생 빈도로 추정한다
- 해당 기능이 트리거하는 도메인 이벤트(생성, 수정, 조회, 삭제)의 예상 빈도
- 평가: 높음(핵심 사용자 플로우) / 중간(보조 플로우) / 낮음(예외 플로우)

### 3. 사용자 가치 (User Value)

- 도메인 모델에서 해당 항목이 핵심 비즈니스 규칙과 얼마나 밀접한가
- 핵심 비즈니스 규칙을 직접 구현하는 항목이 높은 가치
- 평가: 높음(핵심 규칙) / 중간(보조 규칙) / 낮음(편의 기능)

### 4. 기술 부채 (Technical Debt)

- 경계 컨텍스트 위반 정도로 측정한다
- 해당 항목의 구현이 경계 컨텍스트를 깨뜨리거나 불필요한 결합을 만들 리스크
- 평가: 높음(경계 위반 리스크 큼) / 중간(부분적 결합) / 낮음(깔끔한 경계)

## 종합 중요도 산정

4축 평가를 종합하여 중요도 등급을 부여한다:

- **Critical**: 4축 중 3개 이상 "높음"
- **High**: 4축 중 2개 "높음"
- **Medium**: 4축 중 1개 "높음" 또는 3개 이상 "중간"
- **Low**: 나머지

## 산출물 형식

```markdown
# 중요도 분석

## 항목별 중요도

| 항목 | 영향 범위 | 사용 빈도 | 사용자 가치 | 기술 부채 | 종합 |
|------|-----------|-----------|-------------|-----------|------|
| BL-xxx | 높음/중간/낮음 | ... | ... | ... | Critical/High/Medium/Low |

## 중요도별 그룹

### Critical
{항목 목록 및 핵심 근거}

### High
...

### Medium
...

### Low
...
```

## 제약

- 백로그 항목을 수정하지 않는다 (중요도 산정만 수행)
- 모든 항목에 중요도를 반드시 배정한다 (누락 금지)
- 추측이 필요한 경우 "[추론]" 태그를 붙이고 근거를 명시한다
