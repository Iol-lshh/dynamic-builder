# 증거 규격: 문서·소스 인용

명세·요구사항·설계 문서·소스 코드 파일을 근거로 주장할 때 제시해야 하는 증거 포맷.

## 필수 항목

- 파일 경로 (저장소 상대 또는 절대)
- 인용 위치 (라인 번호 또는 섹션 앵커)
- 원문 블록 그대로

## 포맷

```
출처: {path}:{line-range}
(또는 {path}#{section})

> {원문 그대로 인용}
```

여러 블록 인용 시 각 블록의 출처를 개별 표기한다.

## 금지 사항

- "명세에 X 요구가 있다" 등 요약만 제시
- 페이지·섹션 참조 없이 "어딘가에 있다"
- 의역·재구성한 문장
- 라인 번호 없이 파일명만 제시
