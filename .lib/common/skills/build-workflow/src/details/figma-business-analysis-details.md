# figma-business-analysis-details.md

스텝: analyze-business
에이전트: domain-analyst

---

## 목적

Figma 화면 기획서 데이터를 비즈니스 로직 관점에서 분석하여 백로그 항목 형태의 산출물을 생성한다.

## 입력

- `figma-data.json`: 전처리된 Figma 노드 트리

## 분석 관점 (3축)

### 1. 도메인 모델 분석

- 화면에 표현된 도메인 엔티티(사용자, 주문, 상품 등) 식별
- 엔티티 간 관계(1:N, N:M, 소유/참조) 추론
- 텍스트 노드의 라벨/플레이스홀더에서 필드명과 데이터 타입 추출

### 2. 비즈니스 규칙 분석

- 입력 유효성 검증 규칙 추론 (필수 필드, 형식 제약, 범위 제한)
- 조건부 표시/숨김 로직 추론 (variant/state 기반)
- 권한/역할 기반 접근 제어 힌트 식별

### 3. 데이터 흐름 분석

- 화면에서 발생하는 CRUD 작업 식별
- API 호출이 필요한 지점 추론 (목록 조회, 상세 조회, 생성, 수정, 삭제)
- 폼 제출 시 데이터 변환/매핑 로직 추론

## 산출물 형식

각 발견 사항을 백로그 항목 형태로 기술한다.

```markdown
## [분석 축] 항목 제목

- **유형**: 도메인 모델 / 비즈니스 규칙 / API / 데이터 흐름
- **설명**: 발견 사항 1-2문장
- **근거**: figma-data.json 내 참조 노드 (노드 name 또는 id)
- **구현 힌트**: 백엔드/프론트엔드 구현 시 고려할 사항
```

## 제약

- UI/UX 관점(인터랙션, 정보구조, 상태전이, 컴포넌트)은 분석하지 않는다 (analyze-uiux의 범위)
- 추측이 필요한 경우 "[추론]" 태그를 붙이고 근거를 명시한다
