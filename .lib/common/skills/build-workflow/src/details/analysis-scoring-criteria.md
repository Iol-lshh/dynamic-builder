# 분석 문서 채점 기준 (analysis-scoring-criteria)

## 목적

분석/영향도/통합 분석 문서의 품질을 5개 항목으로 채점하여 100점 만점 점수를 산출한다.

적용 대상: impact analysis, coverage analysis, dependency analysis 등 "코드/변경/상태를 관찰·해석"하는 산출물.

---

## 채점 항목

### 1. 범위 완전성 (25점)

분석 대상 범위를 누락 없이 다뤘는가.

| 점수 | 기준 |
|------|------|
| 22~25 | 요구된 대상 파일·영역·시나리오 전부 분석. 누락 없음 |
| 17~21 | 주요 대상 다룸. 주변부 1~2건 누락 |
| 10~16 | 일부 영역만 분석. 중요 대상 누락 다수 |
| 0~9 | 대상 범위가 임의적. 분석되지 않은 영역이 분석된 영역보다 많음 |

### 2. 근거 구체성 (25점)

파일:라인, 명령 출력, 원문 인용 등 관찰 가능한 근거를 제시하는가.

| 점수 | 기준 |
|------|------|
| 22~25 | 모든 주장이 파일:라인·도구 출력·원문 인용으로 뒷받침됨 |
| 17~21 | 대부분 근거 있음. 일반 서술 1~2건 |
| 10~16 | 근거 없이 서술된 판단 다수 |
| 0~9 | 관찰 아닌 추측·일반론 중심 |

### 3. 영향 식별 정확성 (20점)

식별한 영향이 실제 코드·상태와 일치하며, 놓친 영향이 없는가.

| 점수 | 기준 |
|------|------|
| 18~20 | 식별된 영향이 실제와 일치. 놓친 영향 없음. false positive·negative 모두 적음 |
| 14~17 | 대체로 정확. 경미한 오·누락 1~2건 |
| 10~13 | 영향 식별 오류 또는 누락 다수. 후속 의사결정 위험 |
| 0~9 | 실제와 상반된 판단 또는 핵심 영향 전면 누락 |

### 4. 가정의 명시성 (15점)

숨은 전제·제약·선행 조건을 드러내는가.

| 점수 | 기준 |
|------|------|
| 13~15 | 분석에 영향을 주는 모든 가정·제약·외부 의존이 명시됨 |
| 10~12 | 핵심 가정은 명시. 경미한 암묵 전제 1~2건 |
| 6~9 | 여러 판단이 명시되지 않은 전제 위에 세워짐 |
| 0~5 | 가정이 숨겨져 있어 결론의 신뢰성 판단 불가 |

### 5. 결론의 실행 가능성 (15점)

후속 단계(plan/build)가 이 분석만으로 의사결정 가능한가.

| 점수 | 기준 |
|------|------|
| 13~15 | 결론이 구체적이고 후속 단계 입력으로 그대로 쓰임 |
| 10~12 | 대체로 실행 가능. 경미한 추가 해석 필요 1~2건 |
| 6~9 | 결론이 추상적. 후속 단계에서 분석 재수행 필요 |
| 0~5 | 결론 없음 또는 실행 불가 수준으로 모호 |

---

## 총점 산출

| 항목 | 배점 |
|------|------|
| 범위 완전성 | 25 |
| 근거 구체성 | 25 |
| 영향 식별 정확성 | 20 |
| 가정의 명시성 | 15 |
| 결론의 실행 가능성 | 15 |
| **합계** | **100** |

---

## 산출물 형식

```markdown
## Score: {총점}/100

## 항목별 채점

| 항목 | 점수 | 근거 |
|------|------|------|
| 범위 완전성 (25) | {점수}/25 | {누락된 대상 목록} |
| 근거 구체성 (25) | {점수}/25 | {근거 없는 주장 위치} |
| 영향 식별 정확성 (20) | {점수}/20 | {오·누락 영향} |
| 가정의 명시성 (15) | {점수}/15 | {드러나지 않은 전제} |
| 결론의 실행 가능성 (15) | {점수}/15 | {실행 불가 결론} |

## 개선 필요 사항

- {항목}: {개선 권고 — 구체적 수정 방향}
```

---

## retry 판정

- **score >= 80**: 합격. 다음 단계로 진행
- **score < 80**: 불합격. retry loop 재실행. 이 채점 결과가 다음 회차의 피드백으로 전달됨
