# 빌드 완전성 채점 기준 (build-completeness-scoring-criteria)

## 목적

빌드 산출물(브랜치, 파일 구성, 적용 결과)의 완전성을 5개 항목으로 채점하여 100점 만점 점수를 산출한다.

적용 대상: branch-review, migration-result-review 등 "계획대로 적용됐는가"를 검증하는 산출물. 코드 설계 품질은 `code-level-scoring-criteria`로 별도 평가.

---

## 채점 항목

### 1. 의도 반영 완전성 (25점)

계획이 요구한 파일·변경이 빠짐없이 포함되었는가.

| 점수 | 기준 |
|------|------|
| 22~25 | 계획 대상 파일·라인이 모두 포함됨. 누락 없음 |
| 17~21 | 대부분 반영. 경미한 누락 1~2건 |
| 10~16 | 누락 다수. 후속 빌드 실패 위험 |
| 0~9 | 핵심 변경 누락. 계획과 결과 괴리 |

### 2. 스코프 초과 없음 (20점)

계획 범위를 벗어난 파일·변경이 포함되지 않았는가.

| 점수 | 기준 |
|------|------|
| 18~20 | 계획 범위 내 변경만 존재. diff에 군더더기 없음 |
| 14~17 | 거의 정합. 경미한 스코프 초과 1~2건 |
| 10~13 | 의도하지 않은 파일·변경 다수 혼입 |
| 0~9 | diff가 계획과 구조적으로 불일치 |

### 3. 의존성 무결성 (20점)

import·참조·패키지 선언이 깨짐 없이 연결되는가.

| 점수 | 기준 |
|------|------|
| 18~20 | 모든 import·참조가 해결됨. 미사용/누락 없음 |
| 14~17 | 대체로 정합. 경미한 미사용 import 1~2건 |
| 10~13 | 미해결 참조 또는 누락 import 다수 |
| 0~9 | 의존성 그래프 깨짐. 컴파일 단계에서 실패 |

### 4. 빌드 가능성 (20점)

실제 빌드 명령이 통과하는가 (evidence: 빌드 도구 출력).

| 점수 | 기준 |
|------|------|
| 18~20 | 빌드 명령 통과. 경고 없음. 출력 원문 증거로 첨부 |
| 14~17 | 빌드 통과. 경미한 경고 1~2건 |
| 10~13 | 빌드 실패 또는 주요 경고 다수 |
| 0~9 | 빌드 실패. 증거 없음 또는 실행조차 시도되지 않음 |

### 5. 테스트 통과 가능성 (15점)

기존 테스트와 신규 테스트가 통과하는가 (evidence: 테스트 실행 출력).

| 점수 | 기준 |
|------|------|
| 13~15 | 전체 테스트 통과. 실행 결과 원문 첨부 |
| 10~12 | 대부분 통과. 경미한 실패 1~2건 (원인 분석 포함) |
| 6~9 | 기존 테스트 회귀 또는 신규 실패 다수 |
| 0~5 | 테스트 미실행 또는 광범위 실패 |

---

## 총점 산출

| 항목 | 배점 |
|------|------|
| 의도 반영 완전성 | 25 |
| 스코프 초과 없음 | 20 |
| 의존성 무결성 | 20 |
| 빌드 가능성 | 20 |
| 테스트 통과 가능성 | 15 |
| **합계** | **100** |

---

## 산출물 형식

```markdown
## Score: {총점}/100

## 항목별 채점

| 항목 | 점수 | 근거 |
|------|------|------|
| 의도 반영 완전성 (25) | {점수}/25 | {누락 파일/라인 목록} |
| 스코프 초과 없음 (20) | {점수}/20 | {초과 파일 목록} |
| 의존성 무결성 (20) | {점수}/20 | {미해결 참조 위치} |
| 빌드 가능성 (20) | {점수}/20 | {빌드 출력 증거 — evidence-code-execution 형식} |
| 테스트 통과 가능성 (15) | {점수}/15 | {테스트 출력 증거 — evidence-code-execution 형식} |

## 개선 필요 사항

- {항목}: {개선 권고 — 구체적 수정 방향}
```

---

## retry 판정

- **score >= 85**: 합격. 다음 단계로 진행
- **score < 85**: 불합격. retry loop 재실행. 이 채점 결과가 다음 회차의 피드백으로 전달됨
