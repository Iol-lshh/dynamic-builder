# 계획 채점 기준 (plan-scoring-criteria)

## 목적

실행 계획(git 전략, 적용 순서, 명령 설계)의 품질을 5개 항목으로 채점하여 100점 만점 점수를 산출한다.

적용 대상: branch-plan, pick-plan, migration-plan 등 "어떻게 실행할지"를 구체화한 산출물.

---

## 채점 항목

### 1. 전략 적합성 (25점)

선택한 전략(cherry-pick / checkout / patch / merge 등)이 대상 변경의 성격에 맞는가.

| 점수 | 기준 |
|------|------|
| 22~25 | 파일/변경별로 최적 전략을 선택. 대안 전략 대비 근거 제시 |
| 17~21 | 대체로 적합. 경미한 전략 미스매치 1~2건 |
| 10~16 | 일괄 전략으로 처리해 충돌·누락 위험 다수 |
| 0~9 | 전략 선택 근거 없음. 실행 시 충돌 예상 |

### 2. 명령 구체성 (20점)

명령이 그대로 실행 가능한 수준으로 구체화되었는가.

| 점수 | 기준 |
|------|------|
| 18~20 | 모든 명령에 브랜치·커밋 SHA·파일 경로가 명시됨. 복붙 실행 가능 |
| 14~17 | 대부분 구체. 경미한 변수 미정 1~2건 |
| 10~13 | 명령이 서술형. 실행 전 추가 결정 필요 |
| 0~9 | 명령이 추상적. 실행 가능 수준 아님 |

### 3. 순서 정합성 (20점)

명령 실행 순서가 의존 관계·원자성을 보장하는가.

| 점수 | 기준 |
|------|------|
| 18~20 | 선행-후행 의존 관계 명확. 각 단계가 원자적 |
| 14~17 | 대체로 정합. 경미한 순서 이슈 1~2건 |
| 10~13 | 순서 의존성 고려 부족. 중간 실패 시 상태 깨짐 가능 |
| 0~9 | 순서가 임의적. 실행 중 상태 꼬임 예상 |

### 4. 충돌/리스크 대응 (20점)

예상되는 충돌, 경계 케이스, 실패 지점에 대한 대응이 설계되어 있는가.

| 점수 | 기준 |
|------|------|
| 18~20 | 충돌 지점 사전 식별. 해결 방향·대안 경로 명시 |
| 14~17 | 주요 리스크는 다룸. 경미한 경계 케이스 1~2건 누락 |
| 10~13 | 리스크 식별 부족. 실행 중 예외 상황 대응 미비 |
| 0~9 | 충돌/실패 고려 없음. happy path 전제만 존재 |

### 5. 복구 가능성 (15점)

실행 실패 시 원복·재시도 경로가 있는가.

| 점수 | 기준 |
|------|------|
| 13~15 | 백업 브랜치·중간 커밋·재시도 지점이 계획에 포함됨 |
| 10~12 | 대체로 복구 가능. 일부 단계 롤백 불명확 |
| 6~9 | 실패 시 수동 복구 필요. 복구 절차 누락 |
| 0~5 | 원복 경로 없음. 실패 시 손실 위험 |

---

## 총점 산출

| 항목 | 배점 |
|------|------|
| 전략 적합성 | 25 |
| 명령 구체성 | 20 |
| 순서 정합성 | 20 |
| 충돌/리스크 대응 | 20 |
| 복구 가능성 | 15 |
| **합계** | **100** |

---

## 산출물 형식

```markdown
## Score: {총점}/100

## 항목별 채점

| 항목 | 점수 | 근거 |
|------|------|------|
| 전략 적합성 (25) | {점수}/25 | {전략 선택 근거 평가} |
| 명령 구체성 (20) | {점수}/20 | {추상적 명령 위치} |
| 순서 정합성 (20) | {점수}/20 | {순서 이슈 지점} |
| 충돌/리스크 대응 (20) | {점수}/20 | {누락된 리스크} |
| 복구 가능성 (15) | {점수}/15 | {복구 경로 상태} |

## 개선 필요 사항

- {항목}: {개선 권고 — 구체적 수정 방향}
```

---

## retry 판정

- **score >= 80**: 합격. 다음 단계로 진행
- **score < 80**: 불합격. retry loop 재실행. 이 채점 결과가 다음 회차의 피드백으로 전달됨
