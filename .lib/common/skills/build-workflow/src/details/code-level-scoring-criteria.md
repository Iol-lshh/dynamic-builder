# 코드 레벨 채점 기준 (code-level-scoring-criteria)

## 목적

구현 코드의 품질을 6개 항목으로 채점하여 100점 만점 점수를 산출한다.

---

## 채점 항목

### 1. 의존성 방향 적합도 (20점)

| 점수 | 기준 |
|------|------|
| 18~20 | 저수준 모듈이 고수준(추상)에 의존한다. 역방향 의존 없음. 구체 타입에 대한 직접 결합이 경계에만 존재 |
| 14~17 | 대부분 추상 의존. 역방향 의존 1~2건이지만 지역적 영향 |
| 10~13 | 역방향 의존이 여러 모듈에 분산. 구체 타입 결합이 비즈니스 계층까지 침투 |
| 0~9 | 의존성 방향이 일관되지 않음. 고수준이 저수준에 구조적으로 종속 |

### 2. 레이어 적합도 (15점)

| 점수 | 기준 |
|------|------|
| 13~15 | 기술 레이어(controller/service/repository)와 책임 레이어(domain/application/infrastructure)가 일치. 역방향 호출·레이어 월경 없음 |
| 10~12 | 레이어 규칙 대부분 준수. 경미한 월경 1~2건 |
| 6~9 | 비즈니스 로직이 controller·infrastructure에 누설. 레이어 책임 혼재 |
| 0~5 | 레이어 구분이 형식적. 책임이 자의적으로 배치됨 |

### 3. 객체 책임 분리 적합도 (20점)

SRP, 캡슐화, 불변성, 동일/동등성 규칙.

| 점수 | 기준 |
|------|------|
| 18~20 | 각 객체가 단일 책임을 가진다. 내부 상태는 캡슐화되어 불변 원칙이 지켜지고, equals/hashCode 계약이 일관됨 |
| 14~17 | SRP 대체로 준수. 캡슐화·불변성·동등성 중 1개 항목에서 경미한 누락 |
| 10~13 | 책임 혼재 객체 존재. 가변 상태 노출 또는 동등성 계약 위반 1~2건 |
| 0~9 | 책임이 분리되지 않음. 캡슐화 깨짐 또는 동등성 오류가 광범위 |

### 4. DRY 지향도 (10점)

| 점수 | 기준 |
|------|------|
| 9~10 | 동일 로직이 여러 레이어·클래스에 복제되지 않음. 가드·검증·변환 로직이 단일 지점에 응집 |
| 7~8 | 경미한 중복 1~2건. 리팩터링 부담 낮음 |
| 4~6 | 동일 가드가 N개 레이어에 복제되거나 변환 로직이 산재 |
| 0~3 | 구조적 중복. 한 곳을 고치면 N곳을 함께 고쳐야 하는 상태 |

### 5. 컨벤션 수행도 (15점)

프로젝트 내부 규약 — 표기 규칙, 기존 네이밍 패턴, 디렉토리 구조, 프레임워크 사용법의 일관성.

| 점수 | 기준 |
|------|------|
| 13~15 | 프로젝트 컨벤션을 완전히 따름. 기존 패턴을 일관되게 계승 |
| 10~12 | 대체로 준수. 경미한 일탈 1~2건 |
| 6~9 | 기존 패턴과 충돌하는 작성 방식 다수. 컨벤션 적용이 자의적 |
| 0~5 | 프로젝트 컨벤션 무시. 신규 작성분이 기존과 단절 |

### 6. 의미론적 적합도 (20점)

Idiom + Naming + Semantic + ISP. 3개 하위 축의 근거를 각각 명시한다.

- **Idiom**: 언어·프레임워크가 제공하는 표준 API·관용 표현 활용 (예: `Set.copyOf` vs `stream+Collectors`)
- **Naming**: 식별자가 담는 의미의 적합성 — 과잉·과소·모호 금지
- **Semantic**: 메서드명·경로·시그니처와 실제 동작의 일치, 인터페이스 계약(ISP)의 응집

| 점수 | 기준 |
|------|------|
| 18~20 | 세 하위 축 모두 적합. 표준 API 관용 활용, 이름이 의미를 정확히 담음, 시그니처·경로 semantic 일치, 인터페이스가 응집된 계약 |
| 14~17 | 세 축 중 1개에서 경미한 이탈 1~2건 |
| 10~13 | 세 축 중 2개 이상에서 반복적 이탈. 표준 API 재발명 또는 의미 과잉 네이밍 또는 시그니처-동작 불일치 |
| 0~9 | 관용성·네이밍·의미론 다수 영역에서 구조적 결함. 인터페이스가 뚱뚱하거나 이름↔동작 매핑 파탄 |

## 총점 산출

| 항목 | 배점 |
|------|------|
| 의존성 방향 적합도 | 20 |
| 레이어 적합도 | 15 |
| 객체 책임 분리 적합도 | 20 |
| DRY 지향도 | 10 |
| 컨벤션 수행도 | 15 |
| 의미론적 적합도 | 20 |
| **합계** | **100** |

---

## 산출물 형식

```markdown
## Score: {총점}/100

## 항목별 채점

| 항목 | 점수 | 근거 |
|------|------|------|
| 의존성 방향 적합도 (20) | {점수}/20 | {구체적 근거 — 파일:라인 인용} |
| 레이어 적합도 (15) | {점수}/15 | {구체적 근거} |
| 객체 책임 분리 적합도 (20) | {점수}/20 | {SRP/캡슐화/불변성/동등성 각 축 언급} |
| DRY 지향도 (10) | {점수}/10 | {중복 위치 명시} |
| 컨벤션 수행도 (15) | {점수}/15 | {위배된 컨벤션 명시} |
| 의미론적 적합도 (20) | {점수}/20 | {Idiom/Naming/Semantic 각 축 언급} |

## 개선 필요 사항

- {항목}: {개선 권고 — 파일:라인 + 수정 방향}
```

## retry 판정

- **score >= 85**: 합격. 다음 단계로 진행
- **score < 85**: 불합격. retry loop 재실행. 이 채점 결과가 다음 회차의 피드백으로 전달됨
