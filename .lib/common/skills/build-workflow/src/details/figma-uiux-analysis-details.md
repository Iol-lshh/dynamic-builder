# figma-uiux-analysis-details.md

스텝: analyze-uiux
에이전트: uiux-analyst

---

## 목적

Figma 화면 기획서 데이터를 UI/UX 관점에서 분석하여 백로그 항목 형태의 산출물을 생성한다.

## 입력

- `figma-data.json`: 전처리된 Figma 노드 트리

## 분석 관점 (4축)

### 1. 인터랙션 분석

- 버튼, 링크, 입력 필드 등 인터랙티브 요소 식별
- 각 요소의 예상 동작(클릭, 호버, 포커스, 드래그 등) 추론
- 인터랙션 피드백(로딩, 성공/실패 상태) 필요 여부 판단

### 2. 정보 구조 분석

- 화면 내 정보의 계층 구조(헤더 > 섹션 > 항목) 식별
- 내비게이션 패턴(탭, 브레드크럼, 사이드바 등) 식별
- 정보 밀도와 그루핑 패턴 분석

### 3. 상태 전이 분석

- 화면/컴포넌트의 가능한 상태(default, loading, empty, error, success) 식별
- variant/componentProperties에서 상태 전이 힌트 추출
- 각 상태 간 전이 조건과 트리거 추론

### 4. 컴포넌트 분석

- 재사용 가능한 컴포넌트(COMPONENT, COMPONENT_SET, INSTANCE) 식별
- 컴포넌트의 속성(props), 변형(variants), 슬롯 구조 분석
- 공통 컴포넌트와 화면 고유 컴포넌트 구분

## 산출물 형식

각 발견 사항을 백로그 항목 형태로 기술한다.

```markdown
## [분석 축] 항목 제목

- **유형**: 기능 / UI 상태 / 컴포넌트 / 내비게이션
- **설명**: 발견 사항 1-2문장
- **근거**: figma-data.json 내 참조 노드 (노드 name 또는 id)
- **구현 힌트**: UI 구현 시 고려할 사항
```

## 제약

- 비즈니스 로직(도메인 모델, 규칙, 데이터 흐름)은 분석하지 않는다 (analyze-business의 범위)
- 시각적 디자인(색상, 폰트, 간격)은 분석하지 않는다 (전처리 단계에서 제거됨)
- 추측이 필요한 경우 "[추론]" 태그를 붙이고 근거를 명시한다
