# PR Stack Plan Template

이 문서는 WF1(pr-stack-plan)의 최종 산출물 서식을 정의한다.
WF2(pr-stack-execute)는 이 서식을 입력으로 받아 파싱한다.

---

## 서식 명세

### 1. Meta (필수)

PR 스택 계획의 메타데이터. 테이블 형식으로 작성한다.

```markdown
## Meta
| 항목 | 값 |
|------|-----|
| backup-branch | {백업 브랜치명} |
| base-branch | {기준 브랜치명} |
| total-prs | {PR 총 개수, 정수} |
| created-at | {생성 시각, ISO 8601} |
```

| 필드 | 필수 | 설명 |
|------|------|------|
| backup-branch | 필수 | 전체 변경이 보존된 백업 브랜치명 |
| base-branch | 필수 | PR이 머지되는 기준 브랜치명 |
| total-prs | 필수 | PR 총 개수. PR List 섹션의 PR 수와 일치해야 한다 |
| created-at | 필수 | 계획서 생성 시각 |

---

### 2. PR List (필수)

PR별 상세 정보. PR 수만큼 반복한다. 각 PR은 아래 구조를 따른다.

```markdown
## PR List

### PR #{index}: {주제명}
- **순서**: {index} / {total}
- **의존**: PR #{dep1}, PR #{dep2} (또는 "없음")
- **주제**: {기능/리팩토링/버그수정/...}
- **설명**: {1-2줄 설명}
```

| 필드 | 필수 | 설명 |
|------|------|------|
| index | 필수 | 1-based 정수. 실행 순서와 일치 |
| 주제명 | 필수 | PR의 핵심 주제를 짧게 표현 |
| 순서 | 필수 | "{현재 index} / {total}" 형식 |
| 의존 | 필수 | 선행 PR 목록. 없으면 "없음" |
| 주제 | 필수 | 카테고리: 기능, 리팩토링, 버그수정, 문서, 테스트 등 |
| 설명 | 필수 | PR이 수행하는 변경의 요약 (1-2줄) |

#### 2-1. 파일 목록 (필수)

해당 PR에 포함되는 파일 목록.

```markdown
#### 파일 목록
| 파일 경로 | 변경 유형 | 겹침 PR | 비고 |
|-----------|----------|---------|------|
| src/foo.java | modified | - | |
| src/bar.java | modified | PR #2 | 겹침: 함수 X 수정 |
```

| 컬럼 | 필수 | 설명 |
|------|------|------|
| 파일 경로 | 필수 | base-branch 기준 상대 경로 |
| 변경 유형 | 필수 | added / modified / deleted / renamed |
| 겹침 PR | 필수 | 동일 파일을 수정하는 다른 PR 번호. 없으면 "-" |
| 비고 | 선택 | 겹침이 있을 때 구체적 설명 |

#### 2-2. 의존성 (필수)

해당 PR이 다른 PR에 갖는 의존성 상세.

```markdown
#### 의존성
| 의존 대상 | 유형 | 설명 |
|-----------|------|------|
| PR #1 | 코드 의존 | Bar가 Foo의 인터페이스 변경을 전제 |
```

| 컬럼 | 필수 | 설명 |
|------|------|------|
| 의존 대상 | 필수 | 의존하는 PR 번호. 없으면 행 자체를 "없음" 한 줄로 |
| 유형 | 필수 | 코드 의존 / 파일 의존 / 논리 의존 |
| 설명 | 필수 | 의존 관계의 구체적 내용 |

#### 2-3. 겹침 파일 해소 전략 (조건부 필수)

겹침 PR이 존재하는 파일에 대해서만 작성한다. 겹침이 없으면 섹션 자체를 생략할 수 있다.

```markdown
#### 겹침 파일 해소 전략
| 파일 | 겹치는 PR | 전략 | 설명 |
|------|----------|------|------|
| src/bar.java | PR #1, PR #3 | 선행 PR 우선 | PR #1에서 기본 구조, PR #3에서 추가 |
```

| 컬럼 | 필수 | 설명 |
|------|------|------|
| 파일 | 필수 | 겹침이 발생하는 파일 경로 |
| 겹치는 PR | 필수 | 해당 파일을 수정하는 모든 PR 번호 |
| 전략 | 필수 | 해소 방식: 선행 PR 우선 / 후행 PR 덮어쓰기 / 수동 머지 등 |
| 설명 | 필수 | 전략의 구체적 적용 방법 |

---

### 3. 의존성 그래프 (필수)

PR 간 의존성을 텍스트 기반 방향 그래프로 표현한다.

```markdown
## 의존성 그래프
PR #1 (기반 리팩토링)
  └─► PR #2 (기능 A)
  └─► PR #3 (기능 B)
        └─► PR #4 (통합 테스트)
```

- 화살표(►)는 "선행 -> 후행" 방향
- 독립 PR은 별도 루트로 표현
- 순환 의존성이 있으면 안 된다

---

### 4. 실행 순서 (필수)

PR의 실행(브랜치 생성 및 머지) 순서를 테이블로 정리한다.

```markdown
## 실행 순서
| 순서 | PR | 선행 조건 |
|------|-----|-----------|
| 1 | PR #1 | 없음 |
| 2 | PR #2 | PR #1 머지 |
| 3 | PR #3 | PR #1 머지 |
| 4 | PR #4 | PR #2, PR #3 머지 |
```

| 컬럼 | 필수 | 설명 |
|------|------|------|
| 순서 | 필수 | 1-based 실행 순서 |
| PR | 필수 | PR 번호와 주제 |
| 선행 조건 | 필수 | 실행 전 충족해야 할 조건. 없으면 "없음" |

---

## 파싱 규칙 (WF2용)

WF2가 이 서식을 파싱할 때 따르는 규칙:

1. **Meta 테이블**: `| 항목 | 값 |` 헤더 아래 행에서 key-value 추출
2. **PR List**: `### PR #` 패턴으로 PR 블록 시작을 인식
3. **파일 목록 테이블**: `#### 파일 목록` 아래 테이블에서 파일 경로 추출
4. **의존성 테이블**: `#### 의존성` 아래 테이블에서 의존 관계 추출
5. **겹침 해소 전략**: `#### 겹침 파일 해소 전략` 아래 테이블에서 전략 추출
6. **실행 순서 테이블**: `## 실행 순서` 아래 테이블에서 순서 추출
7. **current-pr-index**: 0-based 인덱스로, `PR #{index}`의 index - 1 에 해당
