# 품질 평가 채점 기준표

## 채점 항목 (100점 만점)

### 1. 문제 재개 가능성 (20점)

이 문서만으로 다른 세션에서 작업을 재개할 수 있는가.

| 점수 | 기준 |
|------|------|
| 20 | 문서만으로 즉시 재개 가능. 현재 상태, 다음 할 일, 주의사항이 명확 |
| 15 | 대부분 재개 가능하나 일부 맥락을 추가 탐색해야 함 |
| 10 | 재개 가능하나 상당한 추가 탐색이 필요 |
| 5 | 재개 방향만 파악 가능, 구체적 행동은 불명확 |
| 0 | 재개 불가능 |

### 2. 산출물 도출 과정 완전성 (20점)

모든 산출물의 도출 과정이 빠짐없이 기록되었는가.

| 점수 | 기준 |
|------|------|
| 20 | 모든 산출물과 도출 과정이 완전히 기록됨 |
| 15 | 주요 산출물은 모두 기록, 사소한 누락 1~2건 |
| 10 | 주요 산출물 일부 누락 또는 도출 과정 불완전 |
| 5 | 산출물 목록만 있고 도출 과정이 대부분 누락 |
| 0 | 산출물 목록 자체가 불완전 |

### 3. 스텝 분류 정확성 (15점)

분류가 7개 카테고리(리소스 획득, 분석, 계획, 구현, 검증, 평가, 실행)에 정확히 매핑되었는가.

| 점수 | 기준 |
|------|------|
| 15 | 모든 스텝이 정확히 분류됨 |
| 10 | 대부분 정확, 분류 오류 1~2건 |
| 5 | 분류 기준은 적용했으나 오류가 3건 이상 |
| 0 | 분류 기준이 일관되게 적용되지 않음 |

### 4. 역방향 요약 논리성 (20점)

최종 결과에서 역추적하는 논리가 일관되고 빈틈이 없는가.

| 점수 | 기준 |
|------|------|
| 20 | 최종 결과부터 시작까지 논리적 연쇄가 완전. 최적 경로 명시 |
| 15 | 역추적 논리가 대체로 일관되나 1~2곳 비약 |
| 10 | 역추적은 시도했으나 중간에 논리적 단절 |
| 5 | 역방향이 아닌 순방향 나열에 가까움 |
| 0 | 역방향 요약이 없음 |

### 5. 스크립트 정합성 (10점)

수집된 스크립트가 실제 실행 순서/결과와 일치하는가.

| 점수 | 기준 |
|------|------|
| 10 | 모든 스크립트가 실행 순서·결과와 일치 |
| 7 | 대부분 일치, 사소한 누락 1~2건 |
| 4 | 주요 스크립트는 있으나 순서 또는 결과 불일치 |
| 0 | 스크립트 목록이 불완전하거나 부정확 |

### 6. 핵심 주의 사항 포함성 (15점)

다음 세션에서 주의해야 할 핵심 사항이 포함되었는가.

| 점수 | 기준 |
|------|------|
| 15 | 실수 가능 지점, 제약 조건, 의사결정 기록이 모두 포함 |
| 10 | 주요 주의사항은 있으나 의사결정 근거 누락 |
| 5 | 주의사항이 있으나 구체성 부족 |
| 0 | 주의사항 섹션이 없거나 형식적 |

## 합격 기준

- **총점 80점 이상**: 합격
- **총점 80점 미만**: 불합격 (retry 대상)

## 감점 사유

- 누락: 기록되어야 할 정보가 빠짐
- 불일치: 서로 다른 섹션 간 정보가 모순
- 모호한 서술: 구체적 파일명/명령어 대신 "관련 파일을 수정했다" 등
- 검증 불가능한 주장: 근거 없이 "잘 동작한다" 등

## score 출력 형식

평가 산출물의 마지막에 아래 형식으로 총점을 명시한다. 오케스트레이터가 이 값을 파싱하여 retry 조건을 판단한다.

```markdown
## Score

| 항목 | 배점 | 득점 |
|------|------|------|
| 문제 재개 가능성 | 20 | {점수} |
| 산출물 도출 과정 완전성 | 20 | {점수} |
| 스텝 분류 정확성 | 15 | {점수} |
| 역방향 요약 논리성 | 20 | {점수} |
| 스크립트 정합성 | 10 | {점수} |
| 핵심 주의 사항 포함성 | 15 | {점수} |
| **총점** | **100** | **{총점}** |

**판정**: 합격 / 불합격
```
