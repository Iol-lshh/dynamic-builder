# figma-dependency-analysis-details.md

스텝: analyze-dependencies
에이전트: domain-analyst

---

## 목적

확정된 백로그 항목 간 의존관계를 domain-analyst 관점(경계 컨텍스트, 공유 자원)에서 식별한다.

## 입력

- `backlog-final.md`: apply-advice의 산출물 (확정된 백로그)

## 분석 관점

domain-analyst의 경계 컨텍스트/공유 자원 렌즈를 활용하여 다음 3가지 의존 유형을 식별한다:

### 1. 선후행 관계

- 항목 A의 구현이 항목 B의 선행 조건인 경우
- 판단 기준: A가 생성하는 도메인 엔티티/API를 B가 참조하는가

### 2. 블로커 관계

- 항목 A가 완료되지 않으면 항목 B를 시작할 수 없는 경우
- 선후행보다 강한 의존: A의 산출물이 B의 입력에 필수

### 3. 공유 자원 관계

- 2개 이상의 항목이 동일한 도메인 엔티티, API 엔드포인트, 또는 컴포넌트를 변경하는 경우
- 병렬 구현 시 충돌 리스크 식별

## 산출물 형식

```markdown
# 의존관계 분석

## 의존관계 맵

| 항목 | 의존 대상 | 유형 | 근거 |
|------|-----------|------|------|
| BL-xxx | BL-yyy | 선후행 / 블로커 / 공유 자원 | {경계 컨텍스트/공유 자원 기반 근거} |

## 의존관계 그래프 (텍스트)

{항목 간 의존 방향을 화살표로 표현}

## 독립 항목

{의존관계가 없는 항목 목록. 병렬 구현 가능.}
```

## 제약

- 백로그 항목을 수정하지 않는다 (의존관계 식별만 수행)
- 의존관계가 없는 항목도 "독립 항목"으로 명시적 기술한다
- 추측이 필요한 경우 "[추론]" 태그를 붙이고 근거를 명시한다
