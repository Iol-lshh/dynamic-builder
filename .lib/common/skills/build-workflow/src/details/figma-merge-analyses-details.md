# figma-merge-analyses-details.md

스텝: merge-analyses
에이전트: scope-reduction-reconciler

---

## 목적

UI/UX 분석과 비즈니스 분석 2개 산출물을 통합하여 중복 없는 단일 백로그 초안을 생성한다.

## 입력

- `analysis-uiux.md`: UI/UX 관점 분석 결과 (백로그 항목 형태)
- `analysis-business.md`: 비즈니스 로직 관점 분석 결과 (백로그 항목 형태)

## 통합 규칙

### 1. 중복 식별 및 병합

- 동일 Figma 노드를 참조하는 항목이 양쪽에 존재하면 하나로 병합한다
- 병합 시 양쪽의 설명, 근거, 구현 힌트를 모두 보존한다
- 병합된 항목의 유형은 두 관점을 모두 반영한다 (예: "기능 + 비즈니스 규칙")

### 2. 항목 구조 통일

각 백로그 항목을 아래 형식으로 통일한다:

```markdown
## BL-{번호}: {항목 제목}

- **출처**: uiux / business / 통합
- **유형**: {유형}
- **설명**: {1-3문장}
- **근거**: {참조 노드}
- **구현 힌트**: {프론트엔드/백엔드 고려사항}
```

### 3. 번호 부여

- BL-001부터 순차 부여한다
- 유형별 그룹핑 없이 발견 순서대로 번호를 매긴다

### 4. 갈등 해소

- 양쪽 분석이 동일 노드에 대해 상충하는 해석을 제시하면:
  - 양쪽 해석을 모두 기술하고 "[갈등]" 태그를 붙인다
  - 어느 쪽도 임의로 기각하지 않는다

## 제약

- 입력에 없는 새로운 항목을 추가하지 않는다
- 입력의 항목을 삭제하지 않는다 (스코프 축소는 apply-advice의 범위)
