# 논픽션 글 출력 형식 가이드 (nf-output-format)

## 목적

논픽션 글 초안의 출력 형식을 정의한다. nf-draft 스텝에서 이 형식을 따른다.

---

## 글 구조

```markdown
# {제목}

## 도입

{독자의 관심을 유도하는 도입부. 문제 제기 또는 핵심 질문 제시.}

## 본문

### {소제목 1}

{논증/서사/해설 단위. 각 단위는 하나의 핵심 주장 또는 관찰을 다룬다.}

### {소제목 2}

{...}

### {소제목 N}

{...}

## 결론

{핵심 논지 요약. 독자에게 남기는 메시지 또는 행동 촉구.}
```

---

## 작성 규칙

### 논증 요소

| 요소 | 규칙 |
|------|------|
| 주장 | 각 소제목 단위에 하나의 명확한 주장 |
| 근거 | 주장마다 최소 1개의 구체적 근거(사례, 데이터, 인용) |
| 반론 | 주요 주장에 대해 예상 반론 1개 이상 다룸 |
| 연결 | 소제목 간 논리적 전환이 명시적 |

### 독자 적합성

| 요소 | 규칙 |
|------|------|
| 어휘 | nf-answer-analysis에서 정의한 독자 수준에 맞춤 |
| 설명 깊이 | 전제 지식 수준에 따라 배경 설명 포함/생략 |
| 톤 | nf-answer-analysis에서 정의한 톤 가이드라인 준수 |

### 형식

| 요소 | 규칙 |
|------|------|
| 제목 | 독자의 관심을 유도하는 구체적 제목 |
| 분량 | nf-answer-analysis에서 정의한 매체별 분량 제약 준수 |
| 단락 | 단락당 3~5문장. 한 단락 = 한 아이디어 |
| 인용 | 인용 시 출처 명시 |

---

## 금지 사항

- nf-answer-analysis에 없는 주장이나 근거를 임의로 추가하지 않는다
- 독자 수준을 무시한 전문 용어 남용 금지
- 결론에서 본문에 없는 새로운 논점을 도입하지 않는다
- 감정적 호소로 논증을 대체하지 않는다
