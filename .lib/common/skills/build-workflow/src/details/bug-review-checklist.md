# 버그 검토 체크리스트

변경된 코드에서 잠재적 버그를 식별하기 위한 검토 관점.

## 검토 항목

### 1. Null / Optional 처리
- NullPointerException 가능성이 있는 경로
- Optional.get() 무조건 호출
- null 반환 후 체이닝 호출

### 2. 분기 조건 오류
- 경계값(boundary) 누락: `<` vs `<=`, `>` vs `>=`
- 논리 연산자 오류: `&&` vs `||` 혼동
- 조건 반전: `!condition` 누락 또는 잘못된 부정

### 3. 예외 처리
- catch 블록에서 예외 무시 (빈 catch)
- 잘못된 예외 타입 catch (너무 넓거나 좁은 범위)
- finally/try-with-resources 미사용으로 인한 리소스 누수

### 4. 상태 변이
- 공유 가변 상태에 대한 동시 접근
- 메서드 인자로 받은 컬렉션 직접 수정
- 루프 내에서 컬렉션 수정 (ConcurrentModificationException)

### 5. 데이터 흐름
- 초기화되지 않은 변수 사용
- 사용되지 않는 반환값
- 변수 섀도잉으로 인한 의도치 않은 참조

### 6. 외부 연동
- API 호출 타임아웃 미설정
- 트랜잭션 경계 밖에서의 DB 조작
- 외부 입력 미검증 (SQL Injection, XSS 등)

## 출력 형식

각 발견 항목에 대해:

| 필드 | 설명 |
|------|------|
| 심각도 | CRITICAL / MAJOR / MINOR |
| 위치 | 파일:라인 |

| 근거 | 왜 버그인지 (재현 시나리오 또는 논리적 근거) |
