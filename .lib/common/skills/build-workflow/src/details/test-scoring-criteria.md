# 테스트 채점 기준 (test-scoring-criteria)

## 목적

작성된 테스트의 품질을 6개 항목으로 채점하여 100점 만점 점수를 산출한다.

---

## 채점 항목

### 1. 케이스 커버리지 적합도 (25점)

happy path, edge case, error path, 경계조건을 실제 요구되는 분기에 맞춰 덮는가.

| 점수 | 기준 |
|------|------|
| 22~25 | 요구 분기·경계·예외 케이스가 모두 덮임. 누락 없음 |
| 17~21 | 주요 분기 커버. 경미한 경계/예외 1~2건 누락 |
| 10~16 | happy path 위주. edge·error 다수 누락 |
| 0~9 | 핵심 분기조차 누락. 구현과 테스트 커버 범위 괴리 |

### 2. 어서션 명확성 (20점)

어떤 동작을 검증하는지 한눈에 드러나고, 기대값이 의미적으로 정확한가.

| 점수 | 기준 |
|------|------|
| 18~20 | 어서션이 검증 대상을 명확히 지목. 기대값이 도메인 의미와 일치. 과잉/과소 검증 없음 |
| 14~17 | 대부분 명확. 과잉 어서션·내부 구현 의존 1~2건 |
| 10~13 | 어서션이 구현 세부에 묶여있거나 기대값이 우회적 |
| 0~9 | 무엇을 검증하는지 불명확. tautology 어서션 다수 |

### 3. 테스트 독립성 (15점)

실행 순서·공유 상태·외부 환경에 영향받지 않는가.

| 점수 | 기준 |
|------|------|
| 13~15 | 순서 무관, 공유 상태 없음. 격리 완전 |
| 10~12 | 경미한 순서 의존 또는 공유 fixture 1~2건 |
| 6~9 | 공유 가변 상태 여러 곳. 순서 의존 다수 |
| 0~5 | 격리 실패. 단독 실행 시 깨짐 |

### 4. 픽스처 설계 (15점)

Arrange 단계의 간결성, 의도 노출, 재사용성.

| 점수 | 기준 |
|------|------|
| 13~15 | 픽스처가 의도를 드러내고 간결. 테스트별로 필요한 것만 준비 |
| 10~12 | 대체로 적정. 불필요한 준비 1~2건 |
| 6~9 | 픽스처 과잉. 테스트 의도 파악이 어려움 |
| 0~5 | 픽스처 구조가 테스트 의미를 압도. 재사용 오용 |

### 5. 가독성 (10점)

test name, given-when-then 구조, 언어적 일관성.

| 점수 | 기준 |
|------|------|
| 9~10 | 테스트명이 시나리오를 서술. given-when-then 구조 명확 |
| 7~8 | 대체로 가독. 네이밍·구조 경미 이탈 1~2건 |
| 4~6 | 테스트명이 구현 이름 복제. 구조 흐트러짐 |
| 0~3 | 명세 불명. 테스트명과 실제 검증 불일치 |

### 6. 비용/안정성 (15점)

flaky 회피, slow 테스트 회피, mock 적정성, 외부 의존 관리.

| 점수 | 기준 |
|------|------|
| 13~15 | flaky 요소 없음. mock이 계약 수준에 맞게 적정. 불필요한 I/O 없음 |
| 10~12 | 경미한 flakiness 위험 또는 mock 과잉 1~2건 |
| 6~9 | 시간·순서·네트워크 의존 다수. mock이 내부 구현에 결합 |
| 0~5 | 반복 실행 시 결과 불안정. 외부 의존 관리 실패 |

---

## 총점 산출

| 항목 | 배점 |
|------|------|
| 케이스 커버리지 적합도 | 25 |
| 어서션 명확성 | 20 |
| 테스트 독립성 | 15 |
| 픽스처 설계 | 15 |
| 가독성 | 10 |
| 비용/안정성 | 15 |
| **합계** | **100** |

---

## 산출물 형식

```markdown
## Score: {총점}/100

## 항목별 채점

| 항목 | 점수 | 근거 |
|------|------|------|
| 케이스 커버리지 적합도 (25) | {점수}/25 | {덮은/누락된 분기 파일:라인} |
| 어서션 명확성 (20) | {점수}/20 | {문제 어서션 위치} |
| 테스트 독립성 (15) | {점수}/15 | {공유 상태·순서 의존 지점} |
| 픽스처 설계 (15) | {점수}/15 | {과잉/부족 픽스처 위치} |
| 가독성 (10) | {점수}/10 | {네이밍·구조 이슈} |
| 비용/안정성 (15) | {점수}/15 | {flaky/slow 요소} |

## 개선 필요 사항

- {항목}: {개선 권고 — 파일:라인 + 수정 방향}
```

---

## retry 판정

- **score >= 80**: 합격. 다음 단계로 진행
- **score < 80**: 불합격. retry loop 재실행. 이 채점 결과가 다음 회차의 피드백으로 전달됨
