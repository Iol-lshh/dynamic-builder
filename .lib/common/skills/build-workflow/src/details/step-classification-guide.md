# 작업 스텝 분류 가이드

## 분류 체계 (7개 행동 스텝)

| 스텝 | 정의 | 예시 |
|------|------|------|
| 리소스 획득 | 파일 읽기, 검색, 외부 정보 수집 | Read, Glob, Grep, WebSearch, git log |
| 분석 | 코드/구조/요구사항 분석, 현황 파악 | 코드 리뷰, 의존성 분석, 요구사항 파싱 |
| 계획 | 설계, 방향 결정, 스텝 분해 | 아키텍처 설계, 구현 계획 수립, 접근법 선택 |
| 구현 | 코드 작성, 파일 생성/수정 | Write, Edit, 코드 생성 |
| 검증 | 테스트 실행, 빌드 확인, lint | npm test, tsc, eslint, 빌드 성공 확인 |
| 평가 | 품질 채점, 리뷰, 비교 | 코드 리뷰 점수, 설계 평가, 대안 비교 |
| 실행 | 명령어 실행, 스크립트 구동, 배포 | npm run, 서버 시작, 스크립트 실행 |

## 분류 원칙

- 각 산출물의 **도출 과정**을 단일 행동 스텝으로 분해한다
- 하나의 도출 과정이 복합 행동이면, **주된 행동**으로 분류한다
- 동일 산출물에 여러 스텝이 관여할 수 있다 (예: 리소스 획득 → 분석 → 구현)

## 출력 형식

```markdown
## 스텝 분류표

### 산출물: {산출물명}

| # | 스텝 | 분류 | 설명 |
|---|------|------|------|
| 1 | 기존 코드 읽기 | 리소스 획득 | 현재 구현 확인 |
| 2 | 패턴 분석 | 분석 | 기존 패턴과 비교 |
| 3 | 새 함수 작성 | 구현 | 분석 결과 반영하여 작성 |
```

## 주의사항

- 스텝의 시간순 나열이 아닌, 산출물 도출에 필요했던 과정의 논리적 순서로 정리한다
- 불필요했던 스텝(시행착오, 되돌린 작업)도 식별하되 별도 표시한다
