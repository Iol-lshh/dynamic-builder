# 성능 검토 체크리스트

변경된 코드에서 성능 저하 요인을 식별하기 위한 검토 관점.

## 검토 항목

### 1. 알고리즘 복잡도
- O(n^2) 이상의 중첩 루프
- 불필요한 전체 탐색 (필터링/인덱싱 가능한 경우)
- 재귀 호출의 깊이 제한 미설정

### 2. 데이터베이스
- N+1 쿼리 문제 (루프 내 개별 쿼리)
- 인덱스 미활용 쿼리 (풀 테이블 스캔)
- 불필요한 SELECT * (필요 컬럼만 조회 가능)
- 대량 데이터 일괄 로딩 (페이징 미적용)

### 3. 메모리
- 대용량 컬렉션 한 번에 메모리 적재
- Stream 미활용 (toList()로 중간 컬렉션 생성)
- 캐싱 가능한 반복 계산

### 4. I/O 및 네트워크
- 동기 블로킹 I/O (비동기 전환 가능 여부)
- 루프 내 반복적 외부 API 호출 (배치 가능)
- 커넥션 풀 미사용 또는 설정 부적절

### 5. 직렬화 / 변환
- 불필요한 객체 변환 체인 (DTO → Entity → DTO)
- 대용량 JSON/XML 직렬화에서 스트리밍 미사용
- toString() 내 무거운 연산

## 출력 형식

각 발견 항목에 대해:

| 필드 | 설명 |
|------|------|
| 영향도 | HIGH / MEDIUM / LOW |
| 위치 | 파일:라인 |
| 설명 | 어떤 성능 문제인지 |
| 근거 | 왜 성능에 영향을 주는지 (복잡도, 호출 빈도 등) |
| 제안 | 개선 방향 및 예상 효과 |
