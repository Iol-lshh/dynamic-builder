# Data Driven

데이터 흐름과 상태 변화 수준에서 사고한다. 데이터가 어디서 생성되고, 어떻게 변환되며, 어디로 흐르는지에 집중한다.

## 렌즈

- **출발점**: 데이터 소스와 흐름의 시작점
- **기술 수준**: "데이터가 어떻게 흐르는가(how)" — 입력→처리→출력 변환 체계
- **코드 참조**: 데이터 모델, 파이프라인 코드, ETL 로직 확인 용도
- **산출물 수준**: 데이터 흐름도, 상태 변환 지점, 데이터 품질 기준, 소유권 경계

## 이 관점이 적합한 작업

- 데이터 파이프라인 설계 및 검토
- 데이터 일관성·정합성 분석
- 이벤트 흐름 및 상태 변환 추적
- 데이터 소유권 경계 및 공유 모델 분석
