# 증거 규격: YAML·설정 파싱

YAML·JSON·TOML 등 구조화된 설정 파일을 근거로 주장할 때 제시해야 하는 증거 포맷.

## 필수 항목

- 파일 경로
- 키 경로 (dot·브래킷 표기, 예: `flow[2].parallel`)
- 해당 영역 원문 블록

## 포맷

````
출처: {path}
키 경로: {a.b[0].c}

```yaml
{원문 블록 그대로}
```
````

## 금지 사항

- "flow가 순차 구조입니다" 등 요약만 제시
- 전체 파일 재구성 또는 의역
- 키 경로 없이 "여기에 설정이 있다"
- 파싱된 값만 제시하고 원문 누락
